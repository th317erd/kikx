# Things that agents need to properly guide themselves to being better

1. A constant reminder that the tests ARE the plan, the todo list, all of it. We MUST stick to Test Driven Development, always.
2. We must have GOOD and BAD testing. Not just happy-paths. Not just failure paths. We should have thorough coverage on happy-paths, failure-paths, and all the grey-area in-between. We NEED to have thorough coverage in our code, and complete confidence in it.
3. The agent almost nearly refuses to write DOM tests for some reason. I constantly remind it, "hey, we CAN and we SHOULD be fully verifying what we need to verify in a unit test, regarding the DOM structure. We have the tools. We can make tests that verify the expected behavior for the DOM. We can use these to our advantage.
4. It can often get similar sounding or similar concepts mixed up. The agent needs EXTRA reminders when things are close together in nature our meaning. For example a "prod_db" and a "local_db" can be a struggle...
