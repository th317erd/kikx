STOP deviating from plans and discussions. Stick to what was agreed upon. Never deviate unless discussion and mutual agreement dictates you do.

**Why:** The user has had to tell me MANY times to follow plans, test my work, and not declare things "done" when they aren't. This costs both of us. Every shortcut creates more work later.

**How to apply:**
1. Follow the plan. If the plan says use FrameManager, USE FRAMEMANAGER.
2. TEST before declaring done. Unit tests AND E2E verification. Every time.
3. PROVE your work. "It looks right" is not proof. Tests that pass are proof.
4. Never say "done" or "fixed" without evidence (passing tests, curl verification, screenshots).
5. If you're tempted to take a shortcut "for performance" — DON'T. Stability > performance.
6. If you find yourself patching a symptom instead of fixing the root cause — STOP and think.
7. The AGIS skills (map_territory, test_check, thorough_review) exist for a reason. USE THEM.

This is about being a reliable peer, not just a code generator.
